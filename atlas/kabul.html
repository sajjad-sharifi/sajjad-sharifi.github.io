Kabul: Health Facility Density

Where do health services concentrate in Afghanistan’s capital?

Author

Sajjad Sharifi

Published

May 25, 2026

Why this analysis

Kabul is Afghanistan’s capital and most populous city, home to roughly 4 million residents. Decades of conflict, displacement, and political transitions have shaped both its built environment and its public services. The central districts in this analysis include the old city, the downtown business district, and the dense residential neighborhoods that grew through successive waves of urbanization.

The same question applies as in Cox’s Bazar: where do health facilities and population actually align, and where do they not? The same methodology, unmodified.

Where are the health facilities?

OpenStreetMap is the source for facility locations. In Kabul, OSM coverage is partial but useful: hospitals, pharmacies, and many clinics are reasonably mapped, often with both Persian and English names. The data is uneven across districts and certainly underrepresents informal or recently-opened facilities. This query captures hospitals, clinics, doctors’ offices, pharmacies, dentists, and anything tagged as healthcare.

# H3 resolution 8 → hexagons roughly 0.5 km across
# (Good resolution for a camp-scale analysis; res 7 = too coarse, res 9 = too fine)
H3_RESOLUTION = 8

# Define our area as an H3 polygon (vertices in lat, lng order)

    (b['south'], b['west']),
    (b['north'], b['west']),
    (b['north'], b['east']),
    (b['south'], b['east']),
]

# Get all H3 cells inside this polygon

# Build a GeoDataFrame where each row is one hexagon
hex_records = []
for h in hex_ids:
    # h3.cell_to_boundary returns list of (lat, lng) tuples
    # Shapely expects (lng, lat) order
    boundary = h3.cell_to_boundary(h)
    poly = Polygon([(lng, lat) for lat, lng in boundary])
    hex_records.append({'h3_id': h, 'geometry': poly})

hexes = gpd.GeoDataFrame(hex_records, crs='EPSG:4326')

# Spatial join: which hex contains each facility?
joined = gpd.sjoin(facilities_pts, hexes, how='left', predicate='within')

# Count facilities per hex
counts = joined.groupby('h3_id').size().rename('facility_count')
hexes = hexes.merge(counts, on='h3_id', how='left')
hexes['facility_count'] = hexes['facility_count'].fillna(0).astype(int)

# Quick stats
print(f"Total hexes:                {len(hexes)}")
print(f"Hexes with ≥1 facility:     {(hexes['facility_count'] > 0).sum()}")

The gap

A hex with zero health facilities matters only if people actually live there. We use OSM building footprints as a population proxy: where structures cluster, residents cluster. Kabul’s central districts are densely built across nearly all of the bbox, so the analytical question becomes which of those densely-built hexes have a facility within roughly 500m.

We classify each 0.5 km hex into one of three categories:

  • Dense, no facility — building count ≥ 50, facility count = 0. The gap zones.
  • Dense, served — building count ≥ 50, facility count ≥ 1. Adequately reached.
  • Low density — fewer than 50 mapped buildings. Outside the dense urban footprint.
Code 
# Classify each hex into one of three categories
def classify_hex(row):
    if row['building_count'] < 50:
        return 'low_density'        # rural / outside camp / empty
    elif row['facility_count'] == 0:
        return 'underserved'        # dense but no facility
    else:
        return 'served'             # dense and at least 1 facility

hexes['access_class'] = hexes.apply(classify_hex, axis=1)

# Summary
print("Hexes by access class:")
print(hexes['access_class'].value_counts())

print(f"\nBuildings in 'underserved' hexes:  "
      f"{hexes[hexes['access_class']=='underserved']['building_count'].sum():>7,}")
print(f"Buildings in 'served' hexes:       "
      f"{hexes[hexes['access_class']=='served']['building_count'].sum():>7,}")

# The headline number
total_dense = hexes[hexes['building_count'] >= 50]['building_count'].sum()
underserved = hexes[hexes['access_class']=='underserved']['building_count'].sum()
pct = 100 * underserved / total_dense
print(f"\n{pct:.1f}% of buildings in dense areas are in 'underserved' hexes.")

What the map shows

The pattern in Kabul is the inverse of Cox’s Bazar. Where the refugee camps had services concentrated in the dense population core, Kabul’s facilities form scattered clusters in a city that is densely built almost everywhere. The result: roughly 74% of mapped buildings sit in hexes with no nearby facility, even though the city has 182 health-tagged features in OSM. Green served zones appear as islands in a sea of red.

That pattern is consistent with rapid urbanization outpacing service provision, which is a common story in fragile-state capitals. The map identifies specific hexes where additional facilities, mobile clinics, or community health workers would expand access most directly.

Limitations

  • OSM facility coverage in Kabul is uneven. The actual service network may be denser than this map suggests, particularly in newer districts where mappers have been less active.
  • Buildings include non-residential structures. OSM building data captures warehouses, government compounds, and embassies alongside dwellings, which inflates the “population” signal in some districts.
  • Presence is not capacity. A pharmacy and a hospital both count as one facility on this map. A refined analysis would weight by type and patient capacity.
  • The bbox is partial. Western Kabul, including some of the most underserved districts historically, falls outside this analysis. Findings apply to the central core.

Methodology

OpenStreetMap features pulled via osmnx. Hexagonal tiling via Uber’s H3 (resolution 8). Spatial joins and aggregation in geopandas. Maps rendered with folium. All code is visible in the cells above (click “Show code” on any block).
