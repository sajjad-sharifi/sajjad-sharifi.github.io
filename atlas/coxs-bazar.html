Cox’s Bazar: Health Facility Density

Where are health services concentrated in the Rohingya refugee settlements?

Author

Sajjad Sharifi

Published

May 25, 2026

Why this analysis

The Rohingya refugee settlements in Cox’s Bazar, Bangladesh form one of the largest concentrations of displaced people in the world. Over 900,000 refugees live across the Kutupalong-Balukhali mega-camp and adjacent sites in Ukhia and Teknaf Upazilas. At that density, in a constrained area, the question of where health services are located relative to where people actually live becomes a basic test of the humanitarian response.

This is the first city in a four-part atlas. The same methodology will be applied without modification to Kabul, Mosul, and Goma. Comparing those four contexts reveals how differently fragility produces spatial patterns of service access. Cox’s Bazar’s pattern, as we will see, is the inverse of Kabul’s.

Where are the health facilities?

# H3 resolution 8 → hexagons roughly 0.5 km across
# (Good resolution for a camp-scale analysis; res 7 = too coarse, res 9 = too fine)
H3_RESOLUTION = 8

# Define our area as an H3 polygon (vertices in lat, lng order)
vertices = [
    (b['south'], b['west']),
    (b['north'], b['west']),
    (b['north'], b['east']),
    (b['south'], b['east']),
]
h3_poly = h3.LatLngPoly(vertices)

# Get all H3 cells inside this polygon
hex_ids = h3.h3shape_to_cells(h3_poly, H3_RESOLUTION)

# Build a GeoDataFrame where each row is one hexagon
hex_records = []
for h in hex_ids:
    # h3.cell_to_boundary returns list of (lat, lng) tuples
    # Shapely expects (lng, lat) order
    boundary = h3.cell_to_boundary(h)
    poly = Polygon([(lng, lat) for lat, lng in boundary])
    hex_records.append({'h3_id': h, 'geometry': poly})

hexes = gpd.GeoDataFrame(hex_records, crs='EPSG:4326')

# Spatial join: which hex contains each facility?
joined = gpd.sjoin(facilities_pts, hexes, how='left', predicate='within')

# Count facilities per hex
counts = joined.groupby('h3_id').size().rename('facility_count')
hexes = hexes.merge(counts, on='h3_id', how='left')
hexes['facility_count'] = hexes['facility_count'].fillna(0).astype(int)

# Quick stats
print(f"Total hexes:                {len(hexes)}")
print(f"Hexes with ≥1 facility:     {(hexes['facility_count'] > 0).sum()}")

The gap

A hex with zero health facilities is only a problem if people live there. We use OSM building footprints — also extensively mapped by HOTOSM in this area — as a proxy for population intensity. A hex with many buildings and zero facilities is the analytical signal we’re after.

We classify each 0.5 km hex into one of three categories:

  • Dense, no facility — building count ≥ 50, facility count = 0. The gap zones.
  • Dense, served — building count ≥ 50, facility count ≥ 1. Adequately reached.
  • Low density — fewer than 50 mapped buildings. Outside the camp footprint, mostly farmland or forest. Not relevant to this analysis.
Code 
# Classify each hex into one of three categories
def classify_hex(row):
    if row['building_count'] < 50:
        return 'low_density'        # rural / outside camp / empty
    elif row['facility_count'] == 0:
        return 'underserved'        # dense but no facility
    else:
        return 'served'             # dense and at least 1 facility

hexes['access_class'] = hexes.apply(classify_hex, axis=1)

# Summary
print("Hexes by access class:")
print(hexes['access_class'].value_counts())

print(f"\nBuildings in 'underserved' hexes:  "
      f"{hexes[hexes['access_class']=='underserved']['building_count'].sum():>7,}")
print(f"Buildings in 'served' hexes:       "

What the map shows

The pattern is unmistakable. Health facilities cluster in a small central core, likely the original Kutupalong camp footprint, where response infrastructure first arrived. That core contains 21 hexes covering roughly 70% of mapped buildings. The periphery, where later refugee inflows settled, is largely unserved: 96 hexes containing nearly 60,000 buildings, with no facility within ~500m.

This is consistent with the way humanitarian response often evolves. Initial services concentrate near where the displacement first appeared, and expansion areas lag behind. The pattern here, services concentrated in the population core, is the opposite of what we find in Kabul, where facilities cluster geographically but the city around them is so dense that most residents fall outside the served zones. Mosul and Goma show yet other variations on the same theme.

Limitations

  • OSM coverage is uneven. Where mappers have been active, as in Cox’s Bazar, the facility list is detailed. In other contexts in this atlas, OSM gaps may understate the actual service presence.
  • Buildings are not population. Some structures are warehouses, learning centres, or sanitation blocks rather than dwellings. The proxy is a rough one.
  • 500m is one choice. Hex resolution H3 res 8 (~0.5 km) is a methodological choice. Larger hexes would smooth out the pattern; smaller ones would amplify it.
  • Access is not presence. A facility within a hex does not guarantee it serves the population there. Capacity, opening hours, fees, and stigma all matter.

Methodology

OpenStreetMap features pulled via osmnx. Hexagonal tiling via Uber’s H3 (resolution 8). Spatial joins and aggregation in geopandas. Maps rendered with folium. All code is visible in the cells above (click “Show code” on any block).
